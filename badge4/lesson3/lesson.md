## 🥋 予期しない文字への対処

多くのデータファイルは **CRLF（キャリッジリターン＋ラインフィード）** をレコードデリミタとして使っています。そのため、別のレコードデリミタを使うファイルでは、CRLF がデータ内に表示されたり読み込まれたりしてしまうことがあります。

SQL では **ASCII 参照** を使ってこれらの文字を扱うことができます：

- **13** → キャリッジリターン（CR）の ASCII コード
- **10** → ラインフィード（LF）の ASCII コード

SQL の `CHR()` 関数（`CHAR()` でも同じ）を使うと、ASCII コード番号で文字を指定できます。

- `chr(13)` → キャリッジリターン
- `chr(10)` → ラインフィード

Snowflake では、`||`（ダブルパイプ）で2つの値を**連結（CONCATENATE）**できるので、CRLF は以下のように表現します：

```
chr(13)||chr(10)
```

`REPLACE` 関数は、置換後の文字を指定しなければ **空文字（何もない状態）に置き換える** ことを覚えておいてください。

---

## 📓 REPLACE() 関数のその他の書き方

以下のように書くこともできます：

```sql
REPLACE($1, chr(13)||char(10))
```

または：

```sql
REPLACE($1, concat(chr(13),chr(10)))
```

または：

```sql
REPLACE($1, '\r\n')
```

SQL ではほとんどの操作に複数の書き方があります。他にもやり方を思いつくかもしれません。

   'DLKW02' as step
