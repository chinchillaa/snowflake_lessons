# 教材の日本語マークダウン変換スキル

## 目的
指定された英語のSnowflake教材ファイルを読み込み、初学者が理解しやすい日本語マークダウン形式のレッスンファイルに変換する。

## 実行手順

1. ユーザーが指定したソースファイル（英語教材）を読み込む
2. 以下の「書式ルール」に従って日本語マークダウンに変換する
3. 変換結果を指定されたパスに書き出す（パスが指定されない場合はソースと同階層の `lesson.md`）

## 書式ルール

### 全体構成
- ドキュメント全体を論理的なセクションに分割し、セクション間は `---`（水平線）で区切る
- 大きなトピックの切り替わりには `# 見出し`（h1）を使う
- セクション内のサブトピックには `## 見出し`（h2）、さらに細かい項目には `### 見出し`（h3）を使う

### 見出しの絵文字プレフィックス
見出しの種類に応じて以下の絵文字を先頭に付与する：
- `📓` — 解説・概念説明のセクション（例: `## 📓 データ型の復習`）
- `🥋` — 手を動かすハンズオン演習（例: `## 🥋 サンプルテーブルを作成してみよう`）
- `🎯` — ユーザーが自力で取り組む課題・チャレンジ（例: `## 🎯 内部ステージを自分で作成する`）
- `🎭` — 概念実証・実験的な内容（例: `## 🎭 概念実証のための迅速なプロトタイピング`）
- `✅` — 確認チェックポイント（例: `### ✅ ラボ完了時の確認`）
- `🏁` — まとめ・次のステップ（例: `# 🏁 まとめと次のステップ`）

### テキスト表現
- 日本語で書く。技術用語は日本語訳の後に括弧で英語原語を併記する（例: `非構造化データ（Unstructured Data）`）
- Snowflakeのオブジェクト名・キーワードは **太字** にする（例: `**SYSADMIN**`、`**VARCHAR**`）
- 概念や用語の初出時は **太字** で強調する
- 文体は「です・ます調」で統一する
- 箇条書きを積極的に活用し、長い段落を避ける
- 比喩やストーリー（登場人物の行動・考え）がある場合は自然な日本語に意訳し、学習の動機づけとして残す

### SQLコードブロック
- SQL文は ` ```sql ` ～ ` ``` ` のフェンスドコードブロックで囲む
- コードブロックの前に、そのSQLが何をするかの簡潔な説明を1〜2行で書く
- コードブロック内のSQLにはコメントを付けない（説明はコードブロックの外に書く）

### テーブル（表）
- 比較・一覧・チェックリストにはマークダウンテーブルを使う
- ヘッダー行と区切り行（`|---|---|`）を含める

### クイズ・確認問題
- 以下の定型フォーマットで記述する：
  ```
  > **続行するには正しい回答が必要です**

  **問題:** 〜？

  - [ ] 選択肢A
  - [ ] 選択肢B
  - [ ] 選択肢C

  **[送信]**
  ```
- 複数選択の場合は「**N個選択してください:**」を問題文の後に追加する

### 引用ブロック
- 重要なポイントや定義は `>` 引用ブロックで囲んで目立たせる

### 補足・注意事項
- 初学者が躓きやすいポイントには、前提知識の簡単な復習を挟む
- 手順が複数ある場合は番号付きリストを使う
- 「過去のワークショップ／レッスンで学んだこと」への参照がある場合は、どのレッスンかを明記する

## 注意事項
- ソースの内容を勝手に省略・追加しない。構成の順序も原則としてソースに従う
- 翻訳ではなく「日本語での再構成」を意識する。直訳調にならないようにする
- ソースにないオリジナルの技術的主張を追加しない
